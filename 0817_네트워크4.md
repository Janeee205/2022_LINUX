# 네트워크4
>
> <h5>Physical Layer (물리계층)</h5>
>
> <h6>인접장비간 적용되는 물리적 protocol (신호 변환 방식, 속도, LAN에 사용되는 connector 및 cable 종류 등)를 정의</h6> 
>
> * 충돌 도메인 : 충돌이 일어날 수 있는 범위
> 브로드캐스트 도메인(Broadcast Domain) : 라우터는 네트워크를 구분 지으며 브로드캐스트가 다른 네트워크에 송신되지 않도록 막으며 브로드캐스트가 도달하는 범위가 브로드캐스트 도메인이다.
> * LAN에서 주로 사용되는 UTP cable은 최대 전송거리가 100M.허브나 리피터를 사용하면 전송거리가 연장.
> * 리피터
>   * cable 전송으로 약화된 신호를 초기화, 증폭, 재전송의 기능을 수행
>   * 리피터와 허브는 상위 계층에서 사용하는 MAC 주소나 IP주소를 이해하지 못하고 단지 전기 신호만을 증폭시키는 역할을 한다.
> * 허브
>   * 리피터와 마찬가지로 전기적 신호를 증폭 
>   * LAN 전송거리를 연장시키고 여러 대의 장비를 LAN에 접속할 수 있도록 한다. (멀티 포트 리피터)
>   * 인텔리젠트 허브와 더미 허브가 있다.  
>   * 허브에 연결된 장비들은 하나의 Collision domain 안에 있다. 
>
> <br>
>
> <h5>Data link Layer (데이터 링크 계층)</h5>
>
> <h6>MAC address를 사용하는 계층</h6>
>
> * LLC (Logical Link Control)
>   * Data link의 부 계층 중 하나로  물리적 매체 상에서 흐름제어와 에러제어 등의 트래픽 관리에 관여
>   * 두 장비간에 link를 설정하고 Frame을 송수신하는 방식과 상위 레이어 protocol의 종류를 알리는 역할  
> * LLC (Logical Link Control)
>   * Frame의 포맷, Ethernet 동작 방식, 충돌감지 및 재전송 방식 등을 정의
> * 브리지 
>   * 브리지, 스위치도 허브와 마찬가지로 Ethernet 장비를 물리적으로 연결하고  Frame의 전송거리를 연장하며 단순히 전기적 신호만을 증폭시키는 것이 아니라 Frame을 다시 만들어서 전송.
>   * 허브와는 달리 Layer 2 주소인 MAC 주소를 보고 Frame 전송 포트를 결정하며 따라서 Layer 2 장비라고 한다. 이제는 브리지 대신 스위치를 사용한다.    
> * 스위치
>   * 브리지와 스위치는 MAC 주소와 해당 장비의 포트번호가 기록된 MAC address table을 보고 목적지에게만 Frame을 전송.
>   * 스위치는 한 포트에서 전송되는 Frame이 MAC address table에 있는 특정 포트로만 전송하기 때문에 다른 포트가 전송하는 Frame과 충돌이 발생하지 않는다.
>   * 스위치는 각각의 포트가 하나의 Collision domain에 있다’고 표현한다. 허브는 1차선 도로, 스위치는 다 차선 도로
>
> <br>
>
> <h5>Network Layer (네트워크 계층)</h5>
>
> <h6>IP 주소(논리적인 주소)를 사용하는 계층</h6>
>
> * 라우터
>   * 라우터와 L3 스위치는  IP주소 등 Layer 3 header에 있는 주소를 참조하여 목적지와 연결되는 포트로 packet을 전송하며 따라서 라우터와 L3 스위치를 Layer 3 장비라고 한다.
>    * 다른 네트워크(LAN) 구간의 장비와 통신을 하려면 반드시 Layer 3 장비를 거쳐야 한다. LAN과 LAN이 통신을 하려면 IP 주소를 보는 L3 장비가 있어야 한다.
>    * 라우터는 특정 인터페이스를 통하여 수신한 packet의 목적지 IP 주소를 보고 목적지와 연결된 인터페이스를 통하여 전송할 것을 결정한다. 이를 Routing이라고 한다.
>    * 라우터의 기본 기능은 경로 결정, 경로에 따른 packet 전송이다
